Overview
- This session started from production debugging context and then shifted heavily into architecture redesign and Mermaid diagram cleanup.
- The target architecture is now Riley-first: Riley is the single front door, uses Sonnet internally for planning and synthesis, and hands execution to Opus.
- Opus is responsible for execution routing, implementation work, and assessing whether the work is complete before anything returns to Riley.

Key architecture decisions captured in docs
- Riley is the single user-facing orchestrator.
- Sonnet is folded into Riley rather than shown as a separate user-facing system.
- Opus is the execution and completion layer.
- Riley may pause in a decisions channel to wait for a user response before handing work to Opus.
- Riley decides how to deliver completion back to the user: voice if the requesting user is still in voice, otherwise by tagging them in groupchat.
- ElevenLabs is the voice relay layer and sits inside the Front Door boundary in the diagrams.
- Independent loops are separate from the front door and separate from agent channels.
- Self improvement engine sits outside the Independent loops box and receives input from the loop boxes.
